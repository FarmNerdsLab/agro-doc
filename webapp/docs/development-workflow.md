# Development Workflow

## Overview

This guide covers best practices for developing, testing, and contributing to the Agro-Doc project.

## Development Environment

### Prerequisites

- Completed [Tutorial 1: Flask Quickstart](./tutorial-1-flask-quickstart.md)
- Completed [Tutorial 2: Running Locally](./tutorial-2-running-locally.md)
- VS Code with Dev Containers extension
- Git configured

### Setup Checklist

- [ ] Repository cloned
- [ ] Dev container running
- [ ] Database initialized (`flask --app flaskr init-db`)
- [ ] App running successfully

## Git Workflow

### Branch Naming
Try to name branches based on what they're addressing, such as "bug-fix", "refactor", "feature", "docs".
